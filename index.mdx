Snif is an AI code review agent built for large repositories. It doesn't just read the diff — it builds a full structural and semantic map of your codebase, so every finding it posts is grounded in real context from your code.

It ships as a single Rust binary. Drop it into any CI pipeline, point it at your repository, and get actionable review comments on every pull request.

## How Snif works

Most code review tools look only at the diff. Snif builds three kinds of context before making a single review call.

<Steps>
  <Step title="Index your repository">
    Run `snif index` to parse every source file with tree-sitter, build a structural relationship graph, generate LLM summaries, and store vector embeddings — all in a local SQLite database. Subsequent runs are incremental.
  </Step>
  <Step title="Retrieve related context">
    When a pull request opens, Snif retrieves related code using three methods in parallel: structural graph traversal (imports, consumers, tests), semantic vector search (similar code elsewhere in the repo), and keyword matching (exact identifier references).
  </Step>
  <Step title="Review with full context">
    The assembled context — diff, changed files, related files, and repository conventions — is sent to your configured LLM. The model reviews with full awareness of how the changed code connects to the rest of the repository.
  </Step>
  <Step title="Filter and publish findings">
    Findings go through aggressive static filtering (confidence, evidence, impact) and an adaptive feedback filter that learns from your team's reactions over time. Survivors are posted as inline PR comments.
  </Step>
</Steps>
