---
title: "Quick start"
description: "Get Snif running on your repository in five steps."
---

<Steps>
  <Step title="Install Snif">
    **Option A: Download a pre-built binary (recommended)**

    Download the latest release for Linux x86_64 from the [releases page](https://github.com/AssahBismarkabah/Snif/releases):

    ```bash
    SNIF_VERSION="3.2.3"
    curl -sSLf "https://github.com/AssahBismarkabah/Snif/releases/download/v${SNIF_VERSION}/snif-x86_64-unknown-linux-gnu.tar.xz" -O
    tar xJf snif-x86_64-unknown-linux-gnu.tar.xz
    mv snif-x86_64-unknown-linux-gnu/snif /usr/local/bin/snif
    ```

    **Option B: Build from source**

    Requires Rust 1.70 or later.

    ```bash
    cargo build --release
    ```

    The binary is placed at `target/release/snif`.
  </Step>

  <Step title="Create a configuration file">
    Create a `.snif.json` file at the root of the repository you want to review. At minimum, set the LLM provider endpoint and model names:

    ```json
    {
      "model": {
        "review_model": "gpt-4o",
        "summary_model": "gpt-4o-mini",
        "endpoint": "https://api.openai.com/v1"
      }
    }
    ```

    The `review_model` is used during review calls. The `summary_model` is used during indexing to generate code summaries — a cheaper, faster model works well here.

    <Note>
      The `.snif.json` file contains no secrets and should be committed to the repository. Secrets are passed via environment variables.
    </Note>
  </Step>

  <Step title="Set your API key">
    Export your LLM provider API key as an environment variable:

    ```bash
    export SNIF_API_KEY=your-api-key
    ```

    Snif checks `SNIF_API_KEY` first, then falls back to `OPENAI_API_KEY`. The key is required for both indexing (to generate summaries) and review.
  </Step>

  <Step title="Index the repository">
    Build the repository index. This parses source files, builds the structural graph, generates LLM summaries, and computes vector embeddings:

    ```bash
    snif index --path /path/to/repo
    ```

    The first run processes every file. Subsequent runs are incremental — only files that have changed since the last index are reprocessed. Use `--full` to force a complete rebuild:

    ```bash
    snif index --path /path/to/repo --full
    ```

    <Tip>
      In CI, cache the `.snif/` directory between runs to take full advantage of incremental indexing and avoid reprocessing unchanged files on every commit.
    </Tip>
  </Step>

  <Step title="Review a code change">
    **From a local diff file:**

    ```bash
    git diff main > /tmp/change.diff
    snif review --path /path/to/repo --diff-file /tmp/change.diff
    ```

    Findings are printed to stdout as JSON.

    **From a GitHub pull request:**

    ```bash
    GITHUB_TOKEN=your-token snif review --repo owner/repo --pr 123
    ```

    When using `--repo` and `--pr`, Snif fetches the diff from GitHub and posts findings as inline review comments on the pull request.
  </Step>
</Steps>
