---
title: '`snif index`'
description: 'Build the repository index by parsing source files, constructing the structural graph, analyzing co-change patterns, generating LLM summaries, and computing vector embeddings.'
---

The `snif index` command prepares the repository for context-aware review. It must be run at least once before using `snif review`. Subsequent runs are incremental by default — only files changed since the last indexed commit are reprocessed.

## Usage

```bash
snif index [--path <PATH>] [--full]
```

## Flags

<ParamField path="--path" type="string" default=".">
  Path to the repository root. Defaults to the current directory.
</ParamField>

<ParamField path="--full" type="boolean" default="false">
  Force a complete rebuild of the index, dropping all existing data. Without this flag, indexing is incremental: Snif reads the git diff since the last indexed commit and reprocesses only changed files.
</ParamField>

## What gets built

Each run of `snif index` executes these steps in order:

1. **Structural graph** — source files are parsed with tree-sitter to extract symbols, imports, and cross-file references. The result is an import graph and symbol index with test-to-source links.
2. **Co-change analysis** — git history is walked to identify file pairs that change together frequently. These co-change correlations inform retrieval during review.
3. **LLM summaries** — each symbol and file receives a short natural-language summary generated by the configured `summary_model`.
4. **Vector embeddings** — summaries are embedded using the AllMiniLML6V2 model (384-dimensional vectors by default). Embeddings power semantic retrieval during review.

## Storage

The index is stored in a SQLite database at `.snif/index.db` by default.

You can change the path with the `index.db_path` field in `.snif.json` or the `SNIF_DB_PATH` environment variable.

<Note>
  The index database is not source code. Add `.snif/` to your `.gitignore` so it is not committed.
</Note>

## Examples

**First-time full index:**

```bash
snif index --full
```

**Incremental update (default):**

```bash
snif index
```

**Index a repository at a custom path:**

```bash
snif index --path /path/to/my-repo
```

**Verbose output to inspect what is happening:**

```bash
RUST_LOG=info snif index --full
```

## Notes

- `summary_model` must be set in `.snif.json` for LLM summaries to be generated. Without summaries, vector embeddings are skipped.
- Languages parsed by default: Rust, TypeScript, Python, Java. Configure `index.languages` in `.snif.json` to add or restrict languages.
- Directories excluded by default: `target`, `node_modules`, `vendor`, `.git`, `build`, `.gradle`, `.mvn`. Configure `index.exclude_patterns` to adjust.
- The `SNIF_API_KEY` (or `OPENAI_API_KEY`) environment variable must be set for the summarization step to succeed.
