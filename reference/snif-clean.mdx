---
title: '`snif clean`'
description: 'Remove all local runtime data generated by Snif, including the index database, embedding cache, and feedback store.'
---

The `snif clean` command deletes the `.snif/` directory and the `.fastembed_cache/` directory from the repository root. It does not touch source code or `.snif.json`.

## Usage

```bash
snif clean [--path <PATH>]
```

## Flags

<ParamField path="--path" type="string" default=".">
  Path to the repository root. The command removes runtime data relative to this path. Defaults to the current directory.
</ParamField>

## What gets removed

| Path | Contents |
|------|----------|
| `.snif/` | Index database (`index.db`), feedback store, and any other runtime data written by Snif. |
| `.fastembed_cache/` | Cached embedding model files downloaded by fastembed. |

<Warning>
  This operation is irreversible. After cleaning, you must run `snif index --full` before using `snif review` again.
</Warning>

`.snif.json` is **not** removed. Your configuration is preserved.

## When to use

- The index appears corrupted or `snif review` produces unexpected errors.
- You want to start with a completely fresh index (e.g., after changing `index.languages` or `index.exclude_patterns`).
- You need to free disk space consumed by the index database or embedding cache.

## Examples

**Clean the current repository:**

```bash
snif clean
```

**Clean a repository at a specific path:**

```bash
snif clean --path /path/to/my-repo
```

**Clean then rebuild from scratch:**

```bash
snif clean && snif index --full
```

## Notes

- If neither `.snif/` nor `.fastembed_cache/` exists, the command prints `Nothing to clean.` and exits cleanly.
- The embedding model cache (`.fastembed_cache/`) will be re-downloaded on the next `snif index` run if it has been removed.
